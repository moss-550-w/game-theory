# Claude.md

## 项目概述
本项目旨在构建一个 **博弈论思想演进全景可视化交互应用**，以四大理论支柱（极小极大定理、纳什均衡、逆向归纳均衡、贝叶斯均衡）为核心骨架，呈现博弈论从思想萌芽到现代交叉学科的内在逻辑与历史脉络。应用采用“星云式四象限”主视图布局，支持以理论为中心的双重螺旋深度探索、多分支数学证明路径对比、经典模型交互模拟等功能，适用于学术梳理、教学演示和知识体系自建构场景。

## 项目目标
1. **直观呈现理论版图**：通过四象限矩阵清晰划分博弈论的四个核心场景与对应理论支柱，展示理论之间的继承与批判关系。
2. **深度理解理论演化**：针对每个理论提供独立的历史演进线与数学支撑线，揭示“为何发展、如何证明”的完整叙事。
3. **交互式学习与验证**：内嵌囚徒困境、逆向归纳、混合策略等模拟器，让用户在互动中直观验证均衡逻辑。
4. **学术严谨性保障**：数学证明路径以多分支树形展示，避免固化唯一逻辑；知识库采用结构化数据，便于专家维护与更新。

## 技术架构
- **前端框架**：React 18 + TypeScript 5
- **可视化引擎**：D3.js v7（布局计算与自定义图表），结合 `@visx/visx` 简化 React 集成
- **渲染方式**：SVG（主视图/低节点数场景）+ Canvas（大规模节点证明树）
- **动画**：Framer Motion 10+（声明式布局动画与路径绘制）
- **状态管理**：Zustand（轻量、切片化订阅，支持持久化学习路径）
- **样式**：Tailwind CSS 3 + Radix UI 无样式组件库
- **数据层**：YAML/JSON 结构化知识库，通过 Vite 导入；理论、事件、数学工具、应用场景解耦维护
- **构建工具**：Vite 5
- **测试**：Vitest + React Testing Library + Playwright（E2E）
- **部署**：Vercel（支持预览分支与 CI/CD 自动部署）

## 核心功能模块

### 1. 四象限理论版图
- 横轴：零和 → 非零和；纵轴：静态/完全信息 → 动态/不完全信息
- 四大支柱分别锚定一个象限，边界渐变模糊表示理论交叉
- 支柱间用永久箭头显示继承/推广关系

### 2. 深度探索视图（以任一理论为中心）
- 左侧历史演进线（时间纵轴，非线性）展示前驱、提出、精炼、延伸节点
- 右侧数学支撑线（层次纵轴）展示底层数学基础 → 核心定理 → 证明构造
- 交叉连线精准标注“历史需求—数学工具”映射

### 3. 多分支证明路径树
- 纳什均衡等关键结论展示多种证明路径（角谷、布劳威尔、范美-格利克斯伯格、塔斯基不动点定理等）
- 支持展开/收起，标注适用条件与优劣

### 4. 交互模拟器（全屏独立模块）
- 囚徒困境：双策略切换 + 实时收益矩阵高亮
- 逆向归纳演示器：可收缩博弈树 + 逆向逐步点亮
- 混合策略模拟器：双概率滑杆 + 实时期望收益曲线

### 5. 筛选与透视
- 理论属性标签筛选（合作/非合作、静态/动态、信息状态）
- 应用领域筛选（拍卖、军控、生物学等），同时高亮相关理论与历史节点
- 时间环/应用环可独立开启，提供宏观背景参照

## 关键设计决策
- **放弃传统时间轴+知识图谱双视图**，改用以理论为中心的四象限星云式布局，避免肢解理论的贯通性
- **避免强制自动联动**，采用“按需弹窗+用户主动跳转”模式，降低认知负荷
- **数学路径不固化**，展示多分支证明树，尊重学术史多样性
- **模拟器独立化**，作为全屏模块而非侧边栏嵌入，保障交互体验

## 给 AI 助手的指引
- 当扩展数学证明路径时，务必保持多分支平等呈现，避免暗示唯一“标准证明”
- 添加理论节点时，同步更新其与已有支柱的继承/批判关系，确保图谱逻辑一致
- 模拟器的数学求解应置于 Web Worker 或纯函数中，避免阻塞主线程
- 所有颜色映射需通过色觉障碍模拟验证，关键状态变化提供形状或标签冗余编码
